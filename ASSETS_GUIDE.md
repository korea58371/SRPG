# SRPG WebGL 에셋(리소스) 관리 가이드라인

본 문서는 방대한 이미지 및 사운드 리소스를 처리하는 PixiJS 기반 WebGL SRPG 프로젝트의 에셋 파이프라인 표준을 정의합니다.

## 1. 디렉토리 구조 (Directory Structure)
모든 동적 리소스(런타임에 게임이 로드하는 큰 용량의 에셋)는 빌드 용량 폭발을 막고 브라우저 스트리밍 캐싱을 활용하기 위해 `src/` 폴더가 아닌 **`public/assets/`** 하위에서 관리해야 합니다.

```text
public/assets/
  ├── ui/
  │   ├── terrain/   : 전략 맵 위에 그려질 산맥, 숲, 성, 항구 등의 지형 스프라이트
  │   └── icons/     : 자원(금화, 식량), 상태이상 아이콘, 병과 심볼 등
  ├── portraits/
  │   ├── heroes/    : 네임드 장수/캐릭터 초상화
  │   └── generics/  : 일반병 및 클론 무장 초상화
  ├── bg/            : 이벤트 씬, 전투 진입 시 표시되는 풀사이즈 배경 일러스트
  └── sounds/
      ├── bgm/       : 배경 음악
      └── sfx/       : 스킬/UI 효과음
```

## 2. 에셋 호출 방식 (PixiJS Assets)
React 기반이더라도 맵 안에서는 성능을 위해 DOM의 `<img src="...">` 태그를 쓰지 않으며, PixiJS의 GPU 텍스처를 활용해야 합니다.

### 스프라이트 배치 예시 (코드 단)
현재 `StrategyMapScreen`에서 사용 중인 텍스트 이모지(`⛰️`)를 지형 스프라이트로 교체하는 패턴입니다:

```tsx
import { Sprite } from '@pixi/react';

// 예시: 역-스케일링 렌더가 결합된 지형 스프라이트 투사
<Sprite 
  image="/assets/ui/terrain/mountain.webp" // public 키워드가 생략된 절대 경로 사용!
  x={x} 
  y={y} 
  anchor={0.5} 
  scale={1 / scale} // 텍스트 역-스케일링 기법과 동일하게 적용
/>
```

## 3. 리소스 최적화 및 제작 규칙
1. **차세대 포맷 사용**: 모든 이미지 에셋은 무거운 무압축 PNG 대신 **WebP 포맷 변환을 강력히 권장**합니다. 투명도를 유지하면서도 용량을 절반 이하로 줄여줍니다.
2. **크기 및 규격화**: 
   - 맵 지형 아이콘 류: `64x64` 또는 `128x128` 정방형 캔버스를 유지하세요.
   - 초상화(Portraits): `512x512` 초과 사이즈는 모바일 환경에서 VRAM 과부하를 유발할 수 있습니다.
3. **아틀라스(Atlas) 시트화 (미래 플랜)**: 추후 지형 아이콘 수가 수십 개로 늘어나면 TexturePacker 등을 이용해 하나의 거대한 `.json` 스프라이트 시트로 묶는 것이 'Draw Call(드로우 콜)'을 극적으로 줄여 렌더 성능을 높입니다.
